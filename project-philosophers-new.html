Philosophes

Problème de synchronisation et concurrence - Projet personnel

Détails du projet

Projet Personnel

Objectif du Projet

Implémenter le problème classique du Dîner des Philosophes de Dijkstra en C avec threads POSIX. 5 philosophes, 5 fourchettes, chacun doit penser puis manger. Les défis : éviter deadlock, famine, et race conditions. Démonstration complète de synchronisation, mutex, et algorithmes de coordination.

Le nombre de philosophes est une variable N modifiable au lancement du programme.

Défis Théoriques et Solutions

Défi : Deadlock

Scénario classique : tous philosophes prennent leur fourchette gauche en même temps, puis attendent la droite qui ne viendra jamais.

Solution : Asymétrie - un philosophe prend droite puis gauche, autres prennent gauche puis droite. Ou utiliser monitor pattern avec condition variables.

Défi : Famine

Même sans deadlock, un philosophe peut avoir très peu de chances manger si ses voisins sont agressifs.

Solution : Monitor avec state machine explicit + condition variables per philosopher. Vérifier que tous états doivent progresser.

Défi : Race Conditions

Accès concurrents à fourchettes et variables d'état sans protection. Comportement imprévisible.

Solution : Tous accès à ressources partagées protégés par mutex. Condition variables pour coordination.

Défi : Complexité Temporelle

Déterminer si toutes ces conditions sont réellement satisfaites demande analyse fine. Très difficile à prouver par test seul.

Solution : Simulation longue avec logging détaillé, analyse statistiques, preuve formelle du pattern utilisé.

Défi : Compréhension Théorique

Il faut comprendre mutex, semaphores, condition variables, et patterns de synchronisation.

Solution : Étude intensive documentation POSIX, articles académiques sur Dijkstra, implémentation pas à pas, tests exhaustifs.

// Philosopher routine avec monitor pattern
void *philosopher_routine(void *arg)
{
    int id = *(int *)arg;
    
    while (!g_end_simulation)
    {
        // THINKING
        usleep(random_sleep());
        
        // HUNGRY
        pthread_mutex_lock(&g_mutex);
        g_state[id] = HUNGRY;
        
        // Vérifier si on peut manger
        while (!can_eat(id))
            pthread_cond_wait(&g_cond[id], &g_mutex);
        
        g_state[id] = EATING;
        g_meals[id]++;
        pthread_mutex_unlock(&g_mutex);
        
        // EATING
        usleep(random_sleep());
        
        // DROP FORKS
        pthread_mutex_lock(&g_mutex);
        g_state[id] = THINKING;
        
        // Notifier voisins qu'une fourchette libre
        if (can_eat((id - 1 + N) % N))
            pthread_cond_signal(&g_cond[(id - 1 + N) % N]);
        if (can_eat((id + 1) % N))
            pthread_cond_signal(&g_cond[(id + 1) % N]);
            
        pthread_mutex_unlock(&g_mutex);
    }
    
    return NULL;
}

// Vérifier si philosopher peut manger
int can_eat(int id)
{
    return (g_state[id] == HUNGRY &&
            g_state[(id - 1 + N) % N] != EATING &&
            g_state[(id + 1) % N] != EATING);
}
