Creepy Night

Plateforme événementielle complète - Projet personnel en équipe de 5

Détails du projet

Projet Personnel

Objectif du Projet

Plateforme web complète pour créer, découvrir et réserver des événements thématiques "Creepy Night". Full-stack avec front-end moderne (Vue.js), back-end REST API (Node.js/Express), authentification JWT et système de réservation avancé.

Projet réalisé en équipe de 5 personnes. Complexité majeure : coordination de développement parallèle sans écrasement de travail ou dépendances circulaires. Communication et organisation critiques.

Stack

Technologies Utilisées

Authentification JWT

Inscription/connexion sécurisée, tokens, refresh tokens

Défis Rencontrés et Solutions

Défi : Coordination Équipe 5 Personnes

Projet compliqué car il fallait bien faire en sorte de ne pas se marcher dessus. Chacun travaille sur features différentes (auth, événements, réservations, paiement, profil) mais tout dépend de l'API commune et du schéma MongoDB. Risque : conflits Git massifs, API instable, dépendances cassées.

Solution : Définir API contracts très tôt, utiliser feature branches strictes, code reviews obligatoires, réunions quotidiennes pour clarifier dépendances, mergeur unique pour éviter conflits, tests d'intégration systématiques.

Défi : Synchronisation Données en Temps Réel

Places disponibles se réduisent pendant réservation. Race condition : deux clients réservent dernière place, ou l'un voit nombre périmé. Pas acceptable en production.

Solution : Transactions MongoDB, verrous, vérification final juste avant commit paiement, notification WebSocket pour mise à jour live du nombre places.

Défi : Intégration Stripe Complexe

Gestion webhook, idempotence (client retry), conciliation paiement avec réservation, fallback si paiement échoue.

Solution : Idempotency keys, status machine (pending -> paid -> confirmed), webhook validation, reconciliation job périodique.

Défi : Frontend Vue.js Performant

Liste 10+ événements, filtrage, pagination, lazy-loading images, infinite scroll. Si mal conçu : lag.

Solution : Virtualisation liste, lazy-loading images, pagination côté serveur, caching local, debounce filtres.

Défi : Travail Régulier d'Équipe

5 personnes, agenda chaotique, difficulté de coordonner sessions pair-programming ou code reviews.

Solution : Calendar partagé, plan sprintés (2 semaines), objectifs clairs par sprint, démo hebdo, documentation hebdomadaire.

// Réservation d'un événement avec vérification atomique
router.post('/reservations', authenticate, async (req, res) => {
    const session = await mongoose.startSession();
    session.startTransaction();
    
    try {
        const event = await Event.findById(req.body.eventId).session(session);
        if (!event || event.spots_available <= 0) {
            throw new Error('No spots available');
        }
        
        // Décrémenter les places et créer réservation
        event.spots_available -= 1;
        await event.save({ session });
        
        const reservation = await Reservation.create([{
            user_id: req.user.id,
            event_id: event._id,
            status: 'pending'
        }], { session });
        
        await session.commitTransaction();
        res.json(reservation[0]);
        
    } catch (err) {
        await session.abortTransaction();
        res.status(400).json({ error: err.message });
    } finally {
        session.endSession();
    }
});
